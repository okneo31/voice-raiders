# Voice Raiders — 게임 설계 문서

## 개요

목소리로 플레이하는 온라인 멀티플레이어 RPG 웹 게임.
경매 페이즈에서 아이템을 음성으로 입찰하고, 전투 페이즈에서 역할별 음성 메커닉으로 보스와 싸운다.

- **플랫폼**: 모바일 최적화 웹앱
- **인원**: 2~5인 온라인 팀
- **음성 입력**: STT (Web Speech API) + 소리 감지 (Web Audio API)
- **게임 구조**: 라운드제 (짧은 라운드 반복, 점수 누적)
- **비주얼**: 카툰/일러스트 스타일

## 게임 흐름

```
로비 → [경매 30초 → 전투 60초] × 5라운드 → 최종 보스 → 결과
```

한 판은 약 8~10분 소요.

### 로비

- 방 생성/참가 (방 코드 공유)
- 역할 선택: 전사 / 마법사 / 힐러
- 마이크 테스트 및 볼륨 캘리브레이션
- 시작 골드 지급 (전원 동일)

### 경매 페이즈 (30초)

매 라운드 시작 시 아이템 1~2개가 경매에 등장.

- **입찰 방법**: 금액을 소리내어 외치면 STT로 인식 (예: "50!", "백골드!")
- **동시 입찰**: 같은 타이밍에 여러 명이 외치면 볼륨이 큰 사람 우선
- **패스**: "패스!"라고 말하면 경매 포기 (골드 절약 전략)
- **아이템 종류**: 무기, 방어구, 스킬 스크롤, 포션 등
- **역할 전용 아이템**: 특정 역할만 사용 가능한 강력한 아이템 존재

### 전투 페이즈 (60초)

경매에서 획득한 장비로 보스와 팀 전투.

#### 역할별 음성 메커닉

| 역할 | 입력 방식 | 메커닉 |
|------|-----------|--------|
| **전사** | 볼륨 (크게 외치기) | 볼륨이 클수록 데미지 증가 |
| **마법사** | STT (주문 읽기) | 화면에 표시된 주문을 정확히 읽으면 데미지. 정확도 비례 |
| **힐러** | 볼륨 제어 (속삭임) | 일정 볼륨 이하로 부드럽게 말해야 치유. 볼륨이 낮을수록 치유력 UP |

#### 보스 패턴

- 라운드마다 다른 보스 등장
- 특수 공격 시 "피해!" 외치면 회피 (STT)
- 라운드가 진행될수록 보스 난이도 상승
- 5라운드째는 최종 보스 (강화된 패턴)

### 결과

- 팀 클리어 여부 + 개인별 기여도
- MVP 선정 (가장 높은 데미지/힐링)
- 총 점수 계산 (잔여 골드 보너스 포함)

## 확장 계획 (Phase 2, 3)

### Phase 2: Dungeon Market 요소

- 던전 탐험 모드 추가 (협동 PvE)
- 보물 발견 시 팀 내 경매 (협동 vs 이기심 딜레마)

### Phase 3: Voice Arena 요소

- PvP 팀 대전 모드
- 스킬 빌드 시스템

## 기술 스택

| 영역 | 기술 | 호스팅 |
|------|------|--------|
| 프론트엔드 | React + Vite | Vercel (무료) |
| 백엔드 | Node.js + Socket.io | Render (무료) |
| 음성 인식 (STT) | Web Speech API | 브라우저 내장 (무료) |
| 소리 감지 (볼륨/피치) | Web Audio API | 브라우저 내장 (무료) |

### 기술 선택 이유

- **Web Speech API**: 브라우저 내장 STT, 추가 비용 없음. 한국어 지원.
- **Web Audio API**: 실시간 볼륨/피치 분석 가능. 클라이언트에서 처리하므로 서버 부하 없음.
- **Socket.io**: 실시간 멀티플레이어 통신. 자동 재연결, 룸 관리 내장.
- **Vercel + Render**: 둘 다 무료 티어 제공. 소규모 트래픽에 충분.

## 주요 데이터 흐름

```
[클라이언트 A] ─── 마이크 입력 ───┐
                                    ├─ Web Audio API (볼륨/피치) ─→ Socket.io ─→ [서버]
[클라이언트 B] ─── 마이크 입력 ───┤                                              │
                                    ├─ Web Speech API (STT) ────→ Socket.io ─→   │
[클라이언트 C] ─── 마이크 입력 ───┘                                              │
                                                                                  ▼
                                                                          게임 로직 처리
                                                                                  │
                                                                                  ▼
                                                                    Socket.io broadcast
                                                                          │
                                                              ┌───────────┼───────────┐
                                                              ▼           ▼           ▼
                                                          [Client A] [Client B] [Client C]
                                                          UI 업데이트  UI 업데이트  UI 업데이트
```

## 제약 사항

- Web Speech API는 Chrome/Edge에서 가장 안정적 (Safari/Firefox 제한적)
- 모바일 브라우저에서 마이크 권한 필요
- Render 무료 티어는 비활동 시 슬립 (첫 접속 시 ~30초 대기)
- 동시 음성 처리 시 각 클라이언트에서 개별 처리 후 결과만 서버로 전송
